# Documentation

## Purpose
Architecture documentation, workflow diagrams, legal notes, and system guides for developers and auditors.

## What This Folder Contains
- Architecture decision records
- Workflow documentation
- Legal compliance notes
- API documentation
- Deployment guides

## What This Folder MUST NOT Contain
- Application code
- Configuration files
- Test files

## Key Documents

### Architecture
- System overview and layer responsibilities
- Data flow diagrams
- Integration points

### Legal & Compliance
- Court admissibility requirements
- Chain-of-custody documentation
- Audit trail specifications

### Workflows
- Inspection workflow states and transitions
- Sample lifecycle management
- Prosecution case handling

### API Reference
- Endpoint documentation
- Request/response formats
- Authentication requirements

## Maintenance Notes
- Assume another AI or developer will maintain this system
- Document intent, not just implementation
- Keep diagrams and docs in sync with code
